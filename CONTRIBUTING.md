# Contributing to Awesome Model Routing

Thank you for your interest in contributing!

## Guidelines

- The repository must be related to model routing (LLM routing, AI gateways, inference serving, load balancing, etc.)
- It must have at least 1,000 stars
- It must be actively maintained (updated within the last 2 years)
- Add it to the most relevant category
- Keep descriptions concise and in English

## How to Contribute

1. Fork this repository
2. Add the repository to the correct category in `README.md`
3. Follow the existing format: `| [**owner/repo**](url) | ⭐ stars | \`Language\` | description |`
4. Submit a Pull Request with a brief description

## Quality Standards

- No duplicates
- No archived repositories
- Descriptions should be informative, not promotional
- Projects must be directly related to model routing or AI infrastructure

## Categories

- **LLM Routers & Smart Routing** — Tools that intelligently route requests to different models
- **AI Gateways & Unified APIs** — Unified API gateways for multiple LLM providers
- **LLM Proxy & Load Balancing** — Proxy servers and load balancers for LLM traffic
- **Inference Serving Engines** — High-performance engines for serving LLM models
- **LLM Orchestration Frameworks** — Frameworks for orchestrating multiple LLMs
- **API Management & Distribution** — Platforms for managing LLM API access
- **Cost Optimization & Observability** — Tools for reducing LLM costs and monitoring
- **Research & Benchmarks** — Academic research and benchmarks for routing strategies
